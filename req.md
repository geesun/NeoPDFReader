# neoPdfReader — 需求文档

## 1. 项目目标

构建一个高性能、跨平台（macOS / Ubuntu）的 PDF 阅读器，核心场景：
- 流畅打开 **16000+ 页**的超大 PDF
- 提供**快速全文搜索**能力
- 参考 Adobe Acrobat Reader 的核心功能集，保持轻量

---

## 2. 平台与环境

| 项目 | 要求 |
|------|------|
| 操作系统 | macOS (ARM/x86), Ubuntu 22.04+ |
| 安装方式 | 原生桌面应用，双击/命令行启动 |
| 最低硬件 | 8 GB RAM, 双核 CPU |

---

## 4. 非功能需求

### 4.1 性能

| 指标 | 目标 |
|------|------|
| 冷启动到可交互 | < 1.5s（普通 PDF）|
| 16000 页 PDF 打开到首页可见 | < 2s |
| 翻页响应 | < 100ms |
| 全文搜索（16000 页） | 首次 < 5s（含索引构建），后续 < 1s |
| 内存占用（16000 页 PDF 打开状态） | < 500 MB |
| 缩放/旋转响应 | < 200ms |

### 4.2 稳定性

- 不因文件损坏而崩溃，优雅降级提示
- 后台线程异常不影响 UI 线程

### 4.3 安全性

- 不执行 PDF 内嵌 JavaScript
- 不自动打开外部链接，需用户确认

### 4.4 可维护性

- 模块化架构，渲染/搜索/UI 解耦
- 单元测试覆盖核心模块 > 80%
- CI/CD 自动构建双平台安装包

## 5. 用户交互流程（核心场景）

```
启动应用
  ├─ 拖拽/打开文件 ──▶ 解析PDF元数据 ──▶ 渲染首页 ──▶ 后台索引文本
  │                                          │
  │                                          ├── 翻页/滚动 ──▶ 按需渲染 + LRU缓存
  │                                          ├── Ctrl+F ──▶ 搜索框 ──▶ 查询索引 ──▶ 高亮结果
  │                                          ├── 缩略图/书签 ──▶ 跳转页面
  │                                          └── 缩放/旋转 ──▶ 重新渲染可视区域
  │
  └─ 空窗口（欢迎页 + 最近文件列表）
```
